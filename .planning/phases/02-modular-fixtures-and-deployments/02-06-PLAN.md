## Phase Goal

**As a** show author, **I want to** validate fixtures and adapt logical pools to concrete deployments through atomic impact review, **so that** one show adapts to different rigs without being rebuilt.

**This plan's slice:** A show author can run `golc pool substitute <pool> --from <fixtureA> --to <fixtureB> --out plan.json` to get a capability-diff impact review that names every missing, incompatible, and unsupported capability by severity (never silently approximated), then accept it via `golc pool apply`, revise it by re-running with a different target, or cancel it by discarding the plan.

<objective>
Add semantic fixture substitution (POOL-06/07/08). `internal/substitution/plan.go` diffs two canonical `FixtureDefinition` capability sets — comparing by `CapabilityType` (semantic), never by raw channel position (D-08) — and produces the same `internal/pool.ImpactPlan` shape a pool update produces (D-16: reuse the proven integrity/freshness/atomic-apply mechanism, do not build a second one). Every capability gap is classified with the D-14 severity taxonomy — missing / incompatible / unsupported — surfaced as a warning the author can knowingly accept past, while a true structural error (for example an invalid target fixture) hard-blocks separately. The change is reviewed and accepted/cancelled as a single atomic unit (D-13): revise means re-running the substitution with a different target, never per-item partial accept. `internal/command/pool.go` exposes `pool substitute` as a dry-run that writes a plan `pool apply` then applies.

Purpose: POOL-06/07 are the phase's "swap a fixture model without silent approximation" guarantee — the safety property that lets a show move between rigs with different hardware.
Output: `internal/substitution/plan.go`, tests, and the `pool substitute` route.
</objective>
