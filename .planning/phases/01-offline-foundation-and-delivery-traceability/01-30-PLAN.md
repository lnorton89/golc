<objective>
Close Gap 1 / CR-01 (Critical, blocks LINR-03 and roadmap SC5 first clause, CONTEXT D-18/D-21): rewire the production `linear apply` handler (`runLinearApply`) so it invokes the full exact-plan apply orchestration `apply.RunApply` — which runs `ValidatePlanFreshness` (D-18 staleness rejection) and `ResumePrefix` (D-21 journal resume without replay) — instead of the bare lower-level `apply.Apply`, which has neither protection. Today the safety machinery exists, is fully implemented, and is unit-tested (`TestScopeLinearApplyResume`), but no production code path invokes it, so re-running an already-applied plan file duplicates every not-yet-linked remote Linear object.

Purpose: Make the LINR-03 truth ("preview and run an idempotent reconciliation ... without duplicating retried work") actually true of the shipped production code path, not just of an orphaned unit-tested helper.

Output: `runLinearApply` calling `apply.RunApply`; a new command-level test proving a second apply of the same plan does not create duplicates; loud CLI single-use documentation as the compensating control for the deliberately-retained `ReadByMarker` stub.

## ReadByMarker decision (recorded per task brief)

This plan wires `apply.RunApply` + `apply.LoadJournal` + `CaptureSnapshot` (MANDATORY — this is what flips LINR-03 to verified) and retains the existing `processLinearClient.ReadByMarker` permanent stub, compensated by (a) the now-wired `ValidatePlanFreshness` staleness guard, which loudly rejects a stale re-apply, and (b) explicit single-use documentation in the CLI usage text. Rationale: a real `ReadByMarker` requires first adding a description-search/list action to `tools/linear-sync/src/protocol.ts`'s `Operation` contract — a larger cross-language contract change (out of scope for a bug-fix round). The within-lineage retry case is already covered by the journal, and the stale-replay case is already covered by freshness rejection, so the marginal safety of a real `ReadByMarker` this round is low and it remains a documented follow-up (01-REVIEW.md WR-05). The minimal single-use guard is an acceptable compensating control for this gap-closure round.
</objective>
