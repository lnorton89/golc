## Phase Goal

**As an** Art-Net operator, **I want to** drive a small lighting rig with correct Art-Net 4 frames computed from the deterministic playback engine and independently verify protocol, target, and timing health, **so that** I can run reliable live output without needing the desktop UI.

> Derived from the ROADMAP.md Phase 4 `**Goal:**` line (prose form); restated in User Story form, not invented — identical to the restatement every 04-0N-PLAN.md carries. Run `/gsd mvp-phase 4` only if the ROADMAP goal line itself should be normalized; it does not change this plan.

<objective>
Close 04-VERIFICATION.md **Gap 2**: the pinned interface's lost/degraded status is never surfaced to the operator. `InterfaceManager.Status()`/`Err()` already exist, are correct, and are unit-tested (`TestInterfaceManagerMarkLostTransitionsStatus`), and D-05's never-auto-switch guarantee is real — but no CLI/IPC route ever reads them. `daemon.go`'s `handleStatus`/`statusPayload` carries only Frame+Targets, `runArtnetInterfaceList` only enumerates raw OS candidates with no pinned indicator, and a doc comment in `Run()` falsely claims the loss is "already surface[d] to any status caller."

This plan threads the already-existing `InterfaceManager` status through to the operator: `artnet status` becomes the authoritative pinned-interface status surface (Task 1), and `artnet interface list` additionally annotates which candidate is pinned and its live state when a daemon is reachable, degrading gracefully to the plain candidate list when none is (Task 2). The false comment is corrected.

Task 1 changes the daemon `statusPayload` AND the CLI `artnetStatusPayload` together (the shared `interface` field must be atomic because `strictjson.DecodeStrict` disallows unknown fields — a one-sided change would break the existing plain/watch status decode). Task 2 is a CLI-only slice that reuses the status payload interface field for the `interface list` annotation.

Purpose: ARTN-01 / Success Criterion 1 currently reports PARTIAL — enumeration/pinning/loss-detection are verified, but the operator can never actually SEE that the pinned interface went lost. This plan makes the degraded state visible.
Output: the two production files above plus their tests, leaving 04-01…04-08 untouched. This plan is wave 2: it re-touches `daemon.go`/`command/artnet.go` that 04-08 also edits, so it runs after 04-08 (files-modified overlap → later wave).
</objective>
