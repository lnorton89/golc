## Phase Goal

**As an** Art-Net operator, **I want to** drive a small lighting rig with correct Art-Net 4 frames computed from the deterministic playback engine and independently verify protocol, target, and timing health, **so that** I can run reliable live output without needing the desktop UI.

> Derived from the ROADMAP.md Phase 4 `**Goal:**` line (prose form); restated in User Story form, not invented ŌĆö identical to the restatement every 04-0N-PLAN.md carries. Run `/gsd mvp-phase 4` only if the ROADMAP goal line itself should be normalized; it does not change this plan.

<objective>
Close 04-VERIFICATION.md **Gap 1**: the per-universe "final values" half of ARTN-05 was never wired to the operator surface. `worker.go`'s `tick()` computes a per-universe 512-byte DMX buffer every tick (via `channelmap.Encode`) but discards it after building the outbound ArtDMX packet; nothing in `health.go` / `daemon.go` / `command/artnet.go` carries those bytes to `golc artnet status`. This plan threads that already-computed buffer through the existing lock-free Health publish pipeline to the CLI, so an operator can actually inspect per-universe final values (`--json`, plain, and watch).

This is a single vertical gap-closure thread expressed as two tasks: Task 1 records + publishes the values inside the Health model (unit-proven in isolation), Task 2 surfaces them through the daemon status payload and the CLI. The split is the internal structure of one thread (produce ŌåÆ surface), not horizontal layering across the phase ŌĆö the daemon `statusPayload` and the CLI `artnetStatusPayload` must gain their matching `universes` field together (Task 2) because `strictjson.DecodeStrict` disallows unknown fields, so a one-sided change would break plain/watch decode.

Purpose: ARTN-05 / Success Criterion 4 currently reports a false pass ŌĆö 04-05-SUMMARY.md's coverage table marks this must-have `pass` against `TestArtnetStatusJSONContainsHealthFields`, which only asserts the substrings `"frame"`, `"targets"`, `"OnCadence"` and never any values field. This plan makes the capability real and re-proves it with a test that decodes the actual bytes.
Output: the four production files above plus their tests, leaving 04-01ŌĆ”04-07 untouched.
</objective>
