## Phase Goal

**As an** Art-Net operator, **I want to** drive a small lighting rig with correct Art-Net 4 frames computed from the deterministic playback engine and independently verify protocol, target, and timing health, **so that** I can run reliable live output without needing the desktop UI.

> Derived from the ROADMAP.md Phase 4 `**Goal:**` line (prose form); all three slots present, restated not invented.

<objective>
Stand up the long-lived, standalone-capable process (D-03/D-04) and the local IPC bridge the CLI clients use to configure and inspect it. This is the new architecture the whole phase's operator surface hangs off: engine + worker + interface manager + IPC listener in one headless process, addressable by short-lived `golc artnet ...` clients over a local named pipe that carries the repo's own command.Request/command.Result shapes. After this plan an in-process integration test starts the daemon, dials the pipe, and round-trips a request — the operator surface (Plan 05) then only needs to register routes.

Purpose: D-03/D-04 are locked and have no in-repo precedent; getting the process/IPC boundary right (and loopback-only by default) is the security- and architecture-critical foundation ARTN-04's "independent of the desktop UI" guarantee sits on.
Output: `internal/artnet/ipc/{server,client}.go`, `internal/artnet/daemon.go`, and the one new external dependency `github.com/microsoft/go-winio`.
</objective>
