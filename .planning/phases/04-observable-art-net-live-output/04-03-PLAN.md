## Phase Goal

**As an** Art-Net operator, **I want to** drive a small lighting rig with correct Art-Net 4 frames computed from the deterministic playback engine and independently verify protocol, target, and timing health, **so that** I can run reliable live output without needing the desktop UI.

> Derived from the ROADMAP.md Phase 4 `**Goal:**` line (prose form); all three slots present, restated not invented.

<objective>
The observable heart of the phase: a background worker that reads the engine's newest complete frame on its own ticker, encodes valid ArtDMX per universe, and fans out to enabled unicast targets without ever being backpressured — plus the health model that proves it (frame staleness, target reachability, per-target send outcomes). After this plan, an integration test with a loopback UDP listener standing in for OLA receives real, correctly-addressed ArtDMX packets while a deliberately hung target proves the send loop never stalls. This is the first genuinely observable Art-Net output slice.

Purpose: ARTN-03 (correct on-wire output + refresh cadence) and ARTN-04 (non-backpressuring consumption) are the two hard technical guarantees of the phase; the health model (ARTN-05) is what makes those guarantees inspectable rather than assumed.
Output: `worker.go` + `health.go` in `internal/artnet`.
</objective>

Create `internal/artnet/worker.go` (package `artnet`). Define `Worker` holding: a frame source (an interface satisfied by `*playback.Engine` via `CurrentFrame() *playback.Frame`, so tests can substitute a fake), the configured universes + their enabled targets, the interface manager (for the pinned local bind IP), a per-universe sequence counter map, a `sendTimeout time.Duration`, and the health recorder (Task 2). Add `const workerTickHz = 40` and derive `tickInterval` from it (do not import the engine's tickHz — keep an independent cadence per Assumption A2).

Implement `Start(ctx)`/`Stop()` copying engine.go's lifecycle exactly: `context.WithCancel`, one goroutine, `time.NewTicker(tickInterval)`, `select { case <-ctx.Done(): return; case <-ticker.C: w.tick(w.frames.CurrentFrame()) }`, `defer ticker.Stop()`.

Implement `tick(frame *playback.Frame)` per RESEARCH Pattern 3: for each configured universe, call channelmap `Encode` (pure, in-memory) to build the DMX buffer, then `EncodeArtDMX(w.nextSeq(universe), 0, PortAddress(universe), data)`; on encode error call `health.RecordEncodeError(universe, err)` (GOLC_ARTNET_ENCODE_FAILED) and `continue` (never panic/return, so other universes still tick). For each enabled target of the universe, dispatch a bounded per-target send that sets a write deadline (`w.sendTimeout`) and writes the packet, recording success/error into health via `RecordSend`. Bound concurrency so a persistently slow target cannot pile up goroutines: keep at most one in-flight send per target (skip this tick's send for a target still busy from the previous tick) rather than an unbounded `go func()` per tick.

Open each target's `*net.UDPConn` via `net.DialUDP` bound to the interface manager's pinned local IP (Pitfall 5: bind by local address, not a device-bind socket option). Ensure `Stop()`/ctx cancel closes conns and ends the goroutine.

Create `internal/artnet/worker_test.go`: (a) a loopback listener receives a decodable ArtDMX packet with expected fields; (b) a slow/unroutable target does not reduce the healthy target's received-packet cadence over a short window (ARTN-04); (c) a disabled target receives nothing; (d) sequence per universe increments and never 0; (e) ctx cancel stops the goroutine.
