## Phase Goal

**As an** Art-Net operator, **I want to** drive a small lighting rig with correct Art-Net 4 frames computed from the deterministic playback engine and independently verify protocol, target, and timing health, **so that** I can run reliable live output without needing the desktop UI.

> Derived from the ROADMAP.md Phase 4 `**Goal:**` line (prose form); all three slots present, restated not invented.

<objective>
Add the optional, opt-in node discovery slice (ARTN-02's discovery half): ArtPoll/ArtPollReply in the codec, a bounded discovery scan, and a `golc artnet discover` route that shows compatible nodes as suggestions only. After this plan an operator can ask "what Art-Net nodes are on my pinned interface?" and get a suggestion list — but adding any of them as a live target stays an explicit, per-node action (D-06), and every parsed reply field is bounds-checked because it is untrusted network input.

Purpose: Discovery is the last ARTN-02 capability. Its security posture (suggestion-only, strict parsing) is the point — a spoofed reply must never be able to redirect live output.
Output: ArtPoll codec additions to `packet.go`, `discovery.go`, and the `artnet discover` route.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted network → ArtPollReply parsing | Discovery consumes replies from arbitrary devices that may be spoofed or malformed. |
| discovery results → target configuration | Discovered nodes are suggestions; promotion to a live target must be an explicit operator gate. |
