## Phase Goal

**As an** Art-Net operator, **I want to** drive a small lighting rig with correct Art-Net 4 frames computed from the deterministic playback engine and independently verify protocol, target, and timing health, **so that** I can run reliable live output without needing the desktop UI.

> Derived from the ROADMAP.md Phase 4 `**Goal:**` line (prose form); all three slots present, restated not invented.

<objective>
Turn the daemon into a usable operator surface: the `golc artnet` CLI (D-01) that starts the headless process (`serve`), selects/inspects the interface (`interface list`), configures universes and targets (`configure`), inspects health (`status`, watch + snapshot + `--json`, D-02/ARTN-05), and takes a node offline/online (`target enable|disable`, D-12). After this plan an operator can drive and inspect the whole Art-Net stack from the command line against a running daemon — the phase's core observable capability.

Purpose: Everything built in Plans 01-04 becomes operator-reachable here. This is also where the `artnet` command/test scope is declared so the repo's `golc test --quick --scope artnet` wrapper works.
Output: `internal/command/artnet.go` + its test file.
</objective>

Create `internal/command/artnet.go`. Declare the scope once via `var _ = MustDeclareScope(ScopeRegistration{Scope:"artnet", Summary:"Configure and inspect Art-Net live output."})` (D-01: this is the CLI surface, no GUI). Register routes with package-level `var _ = MustDeclareRoute(...)` following playback.go's idiom.

`artnet serve` (`runArtnetServe`): invokes `artnet.Run(ctx, ...)` to start the long-lived headless daemon in the foreground (D-04 standalone process); this is the one route that is the server, not a client.

`artnet interface list` (`runArtnetInterfaceList`): a client route — dials the daemon and forwards a Request, OR (if no daemon needed for enumeration) calls `artnet.ListCandidateInterfaces()` directly to list candidate interfaces plus the pinned interface status (ARTN-01). Support `--json` per D-02.

`artnet configure` (`runArtnetConfigure`): parse universe/target/interface args using playback.go's two-tier convention — `--flag value` and `--flag=value` both accepted; malformed args are `GOLC_ARTNET_USAGE` with `ExitCode:2`; validate target/interface values with the Plan 02 validators before forwarding, so validated-but-rejected values are domain errors with `ExitCode:1`. Forward the validated config to the daemon over IPC.

For every client route, on a dial failure return `Result{ExitCode:1, Stderr: GOLC_ARTNET_DAEMON_UNREACHABLE}` (Pattern 5), never a hang.

Create `internal/command/artnet_test.go` and declare the `TestScopeArtnet` marker per the repo convention (read internal/command/test.go to match the exact marker-name derivation), so `golc test --quick --scope artnet` resolves. Add arg-parsing tests: malformed flags -> ExitCode 2 / GOLC_ARTNET_USAGE; invalid target value -> ExitCode 1 / GOLC_ARTNET_TARGET_INVALID; a client route with no daemon running -> GOLC_ARTNET_DAEMON_UNREACHABLE.
