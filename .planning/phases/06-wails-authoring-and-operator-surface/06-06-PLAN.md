<phase_goal>
**As a** playback operator, **I want to** run a prepared show through on-screen, keyboard, and constrained generic MIDI controls with independent local safety actions, **so that** I can operate reliably without learning the authoring system and without the frontend ever owning playback.

This slice makes the complete playback workflow reachable on screen (PLAY-01) and by documented keyboard (PLAY-02) — scene switching, layer toggles, BPM entry/tap, and transport — with the keyboard path handled in-webview (never global hotkeys, which are reserved for the safety cluster).
</phase_goal>

<objective>
Fill the `PlaybackService` binding to forward the complete playback workflow to the existing typed command registry (playback/scene/programming), and build the on-screen playback controls (PLAY-01) plus the in-webview keyboard workflow and a documented shortcut reference (PLAY-02). The keyboard layer is window-scoped keydown handling — deliberately NOT the OS-level global-hotkey path, which is reserved for the safety cluster (Pitfall 4).

Purpose: PLAY-01/02 require the full workflow to be reachable both on screen and by keyboard without MIDI hardware. Both surfaces call the same action handlers, which issue typed commands through the existing registry — the frontend gains no new playback authority.

Output: `internal/wails/svc_playback.go` + `PlaybackControls`, `KeyboardShortcuts`, and `useKeyboardWorkflow`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| webview -> PlaybackService binding | Playback intents cross into the typed command registry |
| OS keyboard -> in-webview keydown | Window-scoped key events (distinct from OS-level safety hotkeys) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-15 | Elevation of Privilege | PlaybackService bindings | medium | mitigate | Bindings only build Requests for already-registered command routes; no raw playback/DMX authority is exposed (Task 1). |
| T-06-16 | Denial of Service | global-hotkey overreach | medium | mitigate | Playback keys are window-scoped keydown, never OS-level global hotkeys, avoiding cross-application shortcut conflicts (Pitfall 4). |
| T-06-17 | Tampering | binding argument validation (V5) | low | mitigate | Invalid args surface the registry's existing diagnostic; the command layer validates before mutating. |
</threat_model>
