## Design System

| Property | Value |
|----------|-------|
| Tool | none — shadcn deliberately bypassed (see rationale below) |
| Preset | not applicable |
| Component library | Radix UI Primitives, unstyled (`@radix-ui/react-dialog` 1.1.19 and sibling packages, exact per-package pins per `.planning/research/STACK.md`) |
| Styling approach | Component-scoped CSS Modules + CSS custom properties for brand tokens (`--page`, `--panel`, `--ink`, `--text`, `--accent`, status-color set). No Tailwind, no shadcn theme layer. **[DEFAULT — not locked upstream, but directly follows STACK.md's explicit instruction: "Use unstyled primitives under a GOLC design system. Avoid a theme framework that dictates console density."]** |
| Icon library | `lucide-react` **[DEFAULT — not locked upstream]**. MIT-licensed, tree-shakeable, stroke-based icon set that pairs cleanly with Radix primitives and the brand's technical/mono aesthetic; no icon library was specified in RESEARCH.md or CONTEXT.md. Confirm or override during planning. |
| Font | Archivo (400/500/600/700/800/900) — UI text, labels, headings, display. JetBrains Mono (400/500/600) — technical/numeric readouts (BPM, bar position, MIDI Note/CC values, diagnostic codes). Both from `.planning/brand/GOLC-Brand-Tokens.md` (Google Fonts, locked). |

**Why no shadcn:** `.planning/research/STACK.md`'s Supporting Libraries table explicitly instructs "Use unstyled primitives under a GOLC design system. Avoid a theme framework that dictates console density" for Radix UI Primitives. shadcn ships a fixed Tailwind-driven visual/density system that this instruction directly rules out for a dense, real-time lighting console. The GOLC brand system (`.planning/brand/GOLC-Brand-Tokens.md`) already supplies a complete, locked token set (color, type, motion) to build custom Radix-based components against instead.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, MIDI mapping row internal spacing |
| sm | 8px | Compact element spacing, list row padding |
| md | 16px | Default element spacing, panel internal padding |
| lg | 24px | Section padding, operator surface card padding |
| xl | 32px | Layout gaps between major panels (e.g. authoring view <-> assignment sidebar) |
| 2xl | 48px | Major section breaks (e.g. between authoring workspace and persistent global bar) |
| 3xl | 64px | Page-level spacing; also the fixed touch-target height for safety cluster controls (see exception below) |

Exceptions:
- **Safety cluster controls (Blackout / Revoke Automation / Stop-Release-All, D-13/D-15):** fixed 64px minimum touch-target height per control, well above the 44px accessibility minimum — these are the phase's highest-consequence, must-hit-under-pressure targets and should read as unmistakably larger than any other on-screen control.
- **Icon-only affordances (per-control MIDI Learn button, D-05; Remove-mapping icon button):** 44px minimum touch target even though the visible icon glyph is smaller, per standard touch-target accessibility practice.
- **Live status bar (PLAY-07) and persistent global safety bar (D-13):** fixed height, not built from the standard scale — treat as a locked chrome region (see Typography/Color) rather than a content panel.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | Archivo Regular (400) | 1.5 |
| Label | 14px | Archivo Regular (400) | 1.4 |
| Heading | 20px | Archivo Semibold (600) | 1.2 |
| Display | 28px | Archivo Semibold (600) | 1.2 |
| Technical readout *(brand-locked exception, see below)* | 14px | JetBrains Mono Medium (500) | 1.4 |

Declared weight set: **Archivo Regular (400) + Archivo Semibold (600)** — the two-weight contract required for this phase's UI text and headings.

**Documented exception:** JetBrains Mono at weight 500 is a third, brand-locked weight reserved exclusively for technical/numeric readouts — BPM value, bar position, MIDI Note/CC numbers, diagnostic/error codes (`GOLC_*` strings), and mapping identifiers. This is not a new discretionary weight; it is `.planning/brand/GOLC-Brand-Tokens.md`'s pre-existing two-family system (Archivo for prose/UI, JetBrains Mono for technical values), scoped narrowly so it never substitutes for the Archivo body/heading pair above.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#E4E0D8` light / `#131419` dark (`--page`) | Application canvas background across every view |
| Secondary (30%) | `#F4F1EB` light / `#1E2027` dark (`--panel`) | Cards, operator-surface panels, sidebar, MIDI mapping list rows, persistent global bar background |
| Accent (10%) | `#1B44D9` Signal Blue (`--accent`) | Primary CTA buttons ("Create Operator Surface", "Learn"), active/selected list-row indicator, active scene highlight, focus rings, links |
| Destructive | `#E23A2E` (`--revoked`) | Destructive confirmations only (delete operator surface, remove MIDI mapping) and generic error-state accents |

Accent reserved for: primary CTA buttons, the currently-selected operator surface row, the active scene indicator in the live status bar, focus rings, and the MIDI-Learn-in-progress affordance. Never used for body text, panel backgrounds, or decorative fills.

### Status Vocabulary (semantic, not part of the 60/30/10 split)

`.planning/brand/GOLC-Brand-Tokens.md` defines a fixed six-color status system used only for state chips/indicators — distinct from the accent above and required by PLAY-07 and the safety cluster:

| Status | Value | Usage |
|--------|-------|-------|
| live | `#1B44D9` | "Controlling source" = manual/live in the status bar |
| frame-lock | `#5AC26A` | Output-state chip when frames are healthy/on-cadence |
| armed | `#C8A24B` | Soft-takeover "armed" chip (D-09/D-11); also a general "waiting for confirm" state |
| revoked | `#E23A2E` | Revoke Automation active status chip; shares the Destructive hue by design (both signal danger/stop) |
| blackout | `#17181C` | Blackout-active status chip and the Blackout button's resting fill |
| offline | `#8A887F` | Daemon-unreachable / controller-disconnected status chip |

Applicable state considerations resolved: 13 covered, 3 backstop, 0 unresolved.

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| empty | Operator surface list (list-collection) | ✅ covered | Renders the "No operator surfaces yet" heading + body and a primary "Create Operator Surface" CTA (see Copywriting Contract). |
| empty | MIDI mapping list, per surface/control (list-collection) | ✅ covered | Renders the "No MIDI mappings yet" heading + body (see Copywriting Contract). |
| loading | MIDI Learn capture session (form, D-05 bounded capture) | ✅ covered | Learn button switches to "Listening for MIDI input…" with a visible Cancel affordance while the capture window is open. |
| loading | Operator surface list / MIDI mapping list on initial daemon connect (list-collection) | 🧪 backstop | A skeleton/dim placeholder is expected while the Wails Go host completes its first daemon status fetch; exact skeleton treatment and timing budget not locked — verify with a held-out UI test at implementation. |
| error | MIDI Learn capture — conflicting Note/CC (form, D-06) | ✅ covered | Renders the mapping-conflict copy (see Copywriting Contract); existing mapping is left untouched, new mapping is rejected outright, never silently overwritten. |
| error | MIDI Learn capture — no input received (form) | ✅ covered | Renders the timeout copy (see Copywriting Contract) after the bounded capture window elapses with no message received. |
| error | Operator surface list / MIDI mapping list — daemon unreachable (list-collection) | ✅ covered | Renders the daemon-unreachable copy (see Copywriting Contract); the safety cluster (Blackout/Revoke Automation/Stop-Release-All) remains reachable regardless, per D-13. |
| populated | Operator surface list (list-collection) | ✅ covered | Each row shows surface name, assigned scene/layer/master count, and MIDI-mapping count; the active/selected surface uses the Signal Blue accent. |
| populated | Assigned-items view inside an operator surface, visible-but-locked (list-collection, D-04) | ✅ covered | Assigned items render at full opacity with the Signal Blue selection indicator; unassigned items render at reduced opacity and disabled interaction — always visible, never hidden. |
| populated | MIDI mapping list (list-collection) | ✅ covered | Each row shows control name, Note/CC/channel, and a Remove affordance; soft-takeover-eligible fader mappings additionally show an armed/not-armed chip using the `armed` status color. |
| partial | Assigned-items view (list-collection, D-04) | ✅ covered | Every item renders in exactly one of two defined states (assigned or locked) — there is no third, partially-assigned row state to design for. |
| overflow | Live status bar — active scene, layers, controlling source (static-content, PLAY-07) | ✅ covered | Scene and layer names truncate with ellipsis at a fixed status-bar column width, with full name on hover/tooltip; the persistent bar's height never grows to accommodate long names. |
| overflow | Operator surface list / MIDI mapping list, many rows (list-collection) | 🧪 backstop | Lists scroll within a fixed-height panel rather than growing the window; exact row-count threshold before scrolling engages is not locked — verify against a representative large show during implementation. |
| overflow | Documented keyboard shortcut reference (static-content, PLAY-02) | 🧪 backstop | Panel scrolls or groups shortcuts by category once content exceeds one screen; exact grouping/layout left to implementation. |
| zero-one-many | Operator surface list (list-collection) | ✅ covered | 0 = empty state; 1 = singular "1 operator surface"; many = plural count, same row layout throughout. |
| zero-one-many | MIDI mapping list (list-collection) | ✅ covered | 0 = empty state; 1 = singular "1 mapping"; many = plural count, same row layout throughout. |
| long-text | Operator surface names, scene/layer names — list rows and live status bar (list-collection, nav, static-content) | ✅ covered | All user-authored names truncate with ellipsis at their container's fixed width, with full text available on hover/tooltip — one uniform rule applied across the surface list, mapping list, and live status bar. |
