<phase_goal>
**As a** playback operator, **I want to** run a prepared show through on-screen, keyboard, and constrained generic MIDI controls with independent local safety actions, **so that** I can operate reliably without learning the authoring system and without the frontend ever owning playback.

This plan delivers the constrained-operator-surface data foundation (PLAY-03): the named, individually-assigned surface model, its validation, and the CLI surface that lets an author build and inspect one — the first slice a real user can exercise end-to-end via `golc-project operatorsurface ...`.
</phase_goal>

<objective>
Create `internal/operatorsurface` (the named, individually-assigned constrained-surface model per D-01/D-02/D-03, including the per-surface MIDI mapping storage shape per D-07), wire it into the single revisioned `show.State` document and its validate() entry point, and expose a `golc-project operatorsurface ...` command surface (create/list/assign/unassign/show) with server-side visible-but-locked authorization (D-04, ASVS V4).

Purpose: Every later Wails frontend slice (06-07 surface UI, 06-08 MIDI learn) and the safety/MIDI subsystems persist and authorize against this model. Building it first, CLI-testable, gives downstream plans a stable contract with no GUI dependency.

Output: `internal/operatorsurface/{model,validate}.go` (+tests), extended `internal/show/state.go` (+schema/migrate bump), and `internal/command/operatorsurface.go` (+tests).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| frontend/CLI -> Go command dispatch | Assignment payloads and control selectors are untrusted input crossing into show mutation |
| Go -> SQLite (.golc) | Persisted surface data crosses into durable storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-01 | Tampering / Elevation of Privilege | operatorsurface Authorize (D-04) | high | mitigate | Re-validate every control action against the active surface's assignment set in Go (`Authorize`), never trust a frontend-disabled control; covered by Task 3 test. |
| T-06-02 | Tampering | show.State.validate() referential integrity | medium | mitigate | Reject dangling scene/layer/master refs at Load/Save (Task 2), so a crafted assignment cannot reference or resurrect a deleted item. |
| T-06-03 | Denial of Service | operatorsurface name/ref input (V5) | low | accept | Names are bounded by existing show-document limits; individual-item refs cannot expand to bulk operations (D-03). Rationale: no unbounded expansion path introduced. |
</threat_model>
