## Phase Goal (MVP vertical slice)

**As a** show author, **I want to** configure deployment interfaces and Art-Net universes/targets through on-screen controls, **so that** I can point the rig at the right network output without dropping to the CLI.

Flagged assumptions (Claude's Discretion — no CONTEXT decision covers PLAY-11; recorded, not silent):
- **Client is a thin IPC client:** ArtnetConfigService uses the in-process command registry (`artnet configure`/`interface list`/`target enable`/`status`), which itself validates and forwards to the supervised daemon — it does NOT import internal/artnet or open its own UDP path (mirrors svc_safety.go's daemon-client boundary).
- **Interaction pattern:** a list/table of configured targets (universe, IP, port, enabled) with an inline "add target" row and an interface picker sourced from `artnet interface list`, NOT a wizard — consistent with the dense-console components.
- **Discovery (`artnet discover`) is optional and out of this slice's happy path:** the thinnest real slice is list-interfaces + configure-target + enable/disable + status; a discover-and-suggest affordance may be added later and is not required for PLAY-11's on-screen configuration capability.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| frontend (webview) -> Go host binding | Target/interface configuration payloads cross into command dispatch |
| Go host -> daemon (named-pipe IPC) | Configuration crosses into the daemon that owns Art-Net output |
| daemon -> network (Art-Net UDP) | Configured targets determine outbound UDP destinations |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-30 | Tampering / Input Validation (V5) | Configure target (IP/universe/port) from the frontend | low | mitigate | `artnet configure` runs artnet.ValidateTarget before any daemon forward (T-04-07); the binding uses the same route, so a malformed/out-of-range target is rejected with GOLC_ARTNET_USAGE/GOLC_ARTNET_TARGET_INVALID before reaching the daemon. |
| T-06-31 | Denial of Service | Rapid reconfigure flooding the daemon | low | mitigate | Configuration is a discrete author action (not a hot loop) and target routes are idempotent; no unbounded client-side queue is introduced. |
| T-06-32 | Information Disclosure | interface list exposing all host NICs | low | accept | Same interface enumeration the existing CLI already prints; no new host-network detail is exposed by the desktop surface. |
| T-06-33 | Spoofing | Desktop surface opening a second, unvalidated Art-Net output path | low | mitigate | ArtnetConfigService never imports internal/artnet nor opens UDP; it is a thin IPC client of the one supervised daemon, so there is exactly one output path. |
| T-06-SC | Tampering | npm/pip/cargo installs | low | accept | No new package installs — reuses already-pinned frontend and Go deps; no supply-chain surface added. |
</threat_model>
