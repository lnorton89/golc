## Phase Goal (MVP vertical slice)

**As a** show author, **I want to** program scenes and looks — creating scenes and pointing their base-look, color-theme, chase, and motion layers at reusable looks — through on-screen controls, **so that** I can build a show on screen without dropping to the CLI.

Flagged assumptions (Claude's Discretion — no CONTEXT decision covers PLAY-12; recorded, not silent):
- **Simplified flag subset:** `internal/command/programming.go` exposes a large CRUD surface (rename / reorder / duplicate / delete for themes, presets, chases, motions, scenes, plus the full `programmer set` attribute matrix). This round binds the core authoring path required by PLAY-12 — create scene, create/record each look kind, enable+point the four scene layers, activate a scene, create a blend — and enough `programmer set` depth to record a base-look/color preset. Full rename/reorder/duplicate/delete and the complete attribute-editing matrix remain the CLI's full-fidelity path this round; they are a documented simplified subset, not silently dropped scope.
- **Interaction pattern:** a scene list (each showing its four layers' enabled/ref state) plus look lists (themes/presets/chases/motions), with inline create + a per-layer "enable + choose look" control — NOT a wizard — consistent with the dense-console components.
</objective>
