<phase_goal>
**As a** playback operator, **I want to** run a prepared show through on-screen, keyboard, and constrained generic MIDI controls with independent local safety actions, **so that** I can operate reliably without learning the authoring system and without the frontend ever owning playback.

This plan delivers the local-priority safety and master-level primitive (PLAY-06/08/09): daemon-resident, in-memory overrides the Art-Net worker checks every tick, plus CLI triggers so a user can drive output to zero (or scale it) and prove it takes effect independent of the UI — the reliability spine every later GUI slice depends on.
</phase_goal>

<objective>
Add daemon-resident, in-memory safety and master-level overrides (Blackout, Stop/Release-All, Revoke Automation, Grand Master, group masters) to the existing Art-Net daemon, checked lock-free by the Worker tick immediately before encoding each frame, and expose them through new low-latency IPC routes and CLI commands. Storage is never touched: these are `atomic` flags/levels on the daemon, not show-state mutations (RESEARCH.md Pattern 1; project "storage stays out of the playback path" constraint).

Purpose: PLAY-06/08/09 require these controls to act "through local priority paths that do not wait for UI, script, API, or model work to complete." The enforcement point must be the process that already owns the per-tick Art-Net output (the daemon), not a round trip through persistence. 06-04's OS-level hotkey and 06-05's on-screen buttons both trigger these same routes.

Output: `internal/artnet/safety.go` (+tests), extended `worker.go`/`daemon.go`, and `internal/command/artnet.go` client routes (+tests).
</objective>
