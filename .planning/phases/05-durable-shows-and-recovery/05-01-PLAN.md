<objective>
Replace `internal/show`'s JSON file I/O with a SQLite-backed `.golc` store while keeping the `show.State` domain struct, `validate()`, and every command call site untouched (D-01/D-02/D-03). This is the thinnest end-to-end slice of the phase: the moment `Load`/`Save` move to SQLite, every existing mutating command (`deployment create`, `pool update`, `scene`, etc.) round-trips a complete show through one durable `.golc` file — SHOW-01 and SHOW-02 are delivered, and the recovery-point write half of SHOW-03 rides inside `Save`'s transaction (D-04).

Purpose: Give the project a transactional, durable single-file format (WAL + FULL) as the foundation every later Phase 5 capability (recovery, migration, diagnostics) builds on.
Output: `internal/show/schema.go`, a reimplemented `internal/show/store.go` (`Load`/`Save`/`LoadForRead` + `openStore` helper + `ErrSchemaTooNew`/`ErrSchemaMigrationRequired` sentinels), and `internal/show/store_test.go`.

## Phase Goal (MVP user story)

**As a** show operator, **I want to** save my complete show to one portable `.golc` file and reliably reopen it, **so that** my authoring work is preserved durably without storage ever disturbing live output.
</objective>
