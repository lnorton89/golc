<objective>
Wire the migration engine (Plan 03) into the operator-facing `show open` command as the D-08 automatic-on-open-with-confirmation flow, and enforce the D-10 newer-format refusal across the mutating `show` routes. Opening an older-schema file detects the pending migration and refuses to touch anything until the user explicitly confirms; on confirm it runs the verified-backup -> transactional-migrate -> atomic-replace sequence and opens the migrated show. Opening (for edit) or saving a newer-than-supported file is refused without rewriting it.

Purpose: Complete SHOW-05 end to end — the safe migration mechanics from Plan 03 become a real, guarded operator workflow, and no mutating command can ever silently rewrite a file it does not understand.
Output: extended `internal/command/show.go` (`runShowOpen` migration branch + mutating-route refusal) and its command test covering the migrate-with-confirm / refuse-without-confirm / refuse-newer matrix.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| disk -> internal/command | The on-disk schema_version drives the open handler's migrate-vs-refuse-vs-open decision; a mutating command must never rewrite a file it cannot fully validate. |
| operator confirmation -> destructive migration | The backup+migrate+swap is destructive-adjacent and must be gated behind an explicit token. |
