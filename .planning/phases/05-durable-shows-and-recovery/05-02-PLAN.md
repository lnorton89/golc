## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| disk -> internal/show | Recovery-point blobs read from an opened `.golc` are untrusted until decoded + validated. |
| CLI args -> filesystem | `--show`/`--to` paths are resolved and read/written. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Tampering | `show open`/`AcceptRecoveryPoint` reading recovery blobs | high | mitigate | Every recovery blob passes `strictjson.DecodeStrict` + `validate()` before it can become the working State; an invalid blob is refused, never partially applied. |
| T-05-05 | Information Disclosure | declined recovery-point rows | low | mitigate | `--discard-recovery` runs an explicit `DELETE FROM recovery_points` so declined session data is removed, not merely hidden from the offer (Security row 5); accepted local-only show content on the operator's own machine matches the project's Phase-4 local-disclosure precedent. |
| T-05-04 | Tampering / Information Disclosure | `--to` save-as destination path | medium | mitigate | Resolve via `resolveWritablePath` (the single existing root-relative-vs-absolute rule). |
</threat_model>
