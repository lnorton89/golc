## Phase Goal

**As an** operator, **I want to** drive one authoritative musical clock by entering or tapping a global BPM and choose whether a BPM change keeps musical position or restarts the loop, **so that** every scene, chase, and motion preset advances against the same deterministic tempo.

This slice (03-06) delivers the pure musical clock the engine (03-07) ticks against and the operator-facing BPM controls, reading the `Tempo`/scene flags added by 03-04.

<objective>
Deliver the pure musical-position clock (SCEN-01, D-10), numeric and tap-tempo BPM entry persisted on `show.State.Tempo` (SCEN-02, SCEN-03), and the preserve-position-or-restart epoch recompute (SCEN-08, D-11). All position math is a pure function of monotonic elapsed time (RESEARCH Pattern 1), never an accumulated tick counter.

Purpose: One authoritative clock is what makes SCEN-09's determinism structurally free — the engine (03-07) recomputes position from (bpm, epoch, now) every tick, so adapter delay changes only when, never what.
Output: `internal/playback/clock.go`, `playback` command scope BPM routes.
</objective>
