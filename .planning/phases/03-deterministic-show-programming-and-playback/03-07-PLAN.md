## Phase Goal

**As a** show author/operator, **I want to** run my programmed scenes through a headless engine whose output is the same at a given musical time no matter how slow or unavailable the UI, persistence, scripts, API, or LLM are, **so that** live playback stays reliable and reproducible.

This capstone slice (03-07) assembles the clock (03-06), scene model + fixed-priority reduce (03-04), and programming objects (03-01/02/03) into the real-time engine and proves SCEN-09 determinism mechanically.

<objective>
Deliver the compiler (all-or-nothing Compile, D-06), the pure evaluator (Evaluate -> Frame via fixed-priority reduce plus chase/motion Resolve(position)), and the real-time engine (tick loop, atomic Frame/plan publish, next-bar staged-edit adoption). Prove SCEN-06 (immediate switch adopted at next bar) and SCEN-09 (deterministic time-indexed output under simulated adapter delay/failure).

Purpose: This is the "deterministic playback harness" of success criterion 5 — the pure-position + lock-free-publish design makes SCEN-09 structurally guaranteed and CI-verifiable via a property-style test.
Output: `internal/playback` compile/evaluate/engine/frame, a headless `playback evaluate`/`playback switch` command surface, and the SCEN-09 property test.
</objective>

| Boundary | Description |
|----------|-------------|
| show file -> compiled plan (Compile) | A hand-editable show document crosses into the engine at Compile; all-or-nothing validation rejects any partial/invalid plan before it can reach live output. |
