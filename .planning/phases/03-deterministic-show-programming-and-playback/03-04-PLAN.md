## Phase Goal

**As a** show author, **I want to** assemble a scene from independently enabled base-look, color-theme, chase, and motion layers resolved by a fixed, predictable priority, **so that** my looks combine the same way every time regardless of adapter timing.

This slice (03-04) is the assembly point: it composes the reusable objects from 03-02/03-03 into scenes with the fixed-priority layer reduce that the engine (03-07) evaluates per tick. It also adds the global `Tempo` the clock (03-06) reads.

<objective>
Deliver the scene/layer/blend domain model (SCEN-01 model, SCEN-04, SCEN-05, SCEN-07): scenes with a configured bar-loop length and four independently enabled, independently selectable layers resolved by fixed priority order (D-01/D-02/D-03), reusable blend presets (SCEN-07), the exactly-one-active-scene invariant (SCEN-04), and a global `Tempo` field the clock plan consumes. Persisted as `Scenes`, `BlendPresets`, and `Tempo` on `show.State`, driven by a `scene` command scope.

Purpose: This is the composition layer between reusable programming objects and the real-time engine; the fixed-priority reduce implemented here is the deterministic core of SCEN-05 and, downstream, SCEN-09.
Output: `internal/scene` package (scene/layer/blend), extended `show.State`, `scene` command routes.
</objective>
