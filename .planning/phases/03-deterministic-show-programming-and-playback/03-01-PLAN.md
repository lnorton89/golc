## Phase Goal

**As a** show author, **I want to** program complete tempo-aware looks — selecting fixtures, editing semantic attributes, and assembling reusable themes, presets, chases, motion presets, and bar-based scenes with swappable layers — and run them through a headless deterministic engine, **so that** one show's output stays deterministic even when the UI, persistence, scripts, API, or LLM providers are slow, unavailable, or restarted.

*(Derived from the ROADMAP Phase 3 Goal and the REQUIREMENTS.md user stories "I can assemble tempo-aware scenes from reusable color, chase, motion, and blend components" and "deterministic playback under adapter delay." The ROADMAP `**Goal:**` line is prose, not the `As a / I want / so that` template — this story is grounded in the documented REQUIREMENTS user stories, not invented.)*

This plan (03-01) is the first vertical slice: an author can select fixtures, edit their semantic attributes, and inspect exactly what the programmer will record — the raw material every later slice (themes, presets, chases, scenes) is recorded from.

<objective>
Deliver fixture selection resolution (PROG-01), semantic attribute editing on selected instances (PROG-02), and a programmer inspection surface (PROG-03), persisted as a `Programmer` scratch buffer on `show.State` and driven by a new `golc programmer` command scope.

Purpose: Selection + programmer state is the foundation every subsequent Phase 3 object (theme, preset, chase, motion preset, scene layer) is recorded from — nothing can be authored until an author can pick fixtures and set semantic values on them.
Output: `internal/programming/selection.go`, `internal/programming/programmer.go`, extended `show.State`, and the `programmer` command scope (`set`, `inspect`, `clear`).
</objective>
