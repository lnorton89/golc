<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| committed toolchain config to downloaded executable | Mage archive URLs/digests select executable bytes that later run contributor tasks. |
| archive/install manifest to executable discovery | A filesystem payload is trusted only when the committed checksum and live install inventory both match. |
| Mage CLI target to internal Go command APIs | User-selected target names become command registry routes/arguments without a shell boundary. |
| commands.pr policy to task execution | Committed ordered strings become a serial executable delivery graph with one network-allowed bootstrap node. |
| caller environment/working directory to repository operations | GOLC_PROJECT_ROOT controls every config, build, test, and package path. |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-u0p-02 | Spoofing | `ResolveMageExecutable` | high | mitigate | Require the current strict platform pin, matching live install manifest, regular non-symlink platform executable, and no PATH or ambient fallback. |
