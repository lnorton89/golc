<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| archive source/cache to executable install | Remote or local archive bytes and existing cache files are untrusted until the exact committed SHA-256 and complete archive structure pass. |
| archive metadata to repository filesystem | Entry paths, types, link targets, duplicate names, and permission bits can attempt traversal, overwrite, privilege, or special-file attacks. |
| existing `.tools` state to no-op decision | Manifests and installed bytes may be legacy, partial, corrupt, or attacker-edited; only the current complete schema can suppress acquisition. |
| committed configuration to runtime platform | A Windows archive must never be installed/executed on another runtime, and missing platform pins must never be guessed. |
| bootstrap engine to Go/npm/tsc child processes | Child arguments, working directory, environment, and outputs cross into executable tooling and must avoid shell injection, host-tool fallback, and source-lock mutation. |
| npm registry/cache to linear-sync node_modules | Exact-lock packages are executable supply-chain inputs even with lifecycle scripts disabled; integrity and repeatability depend on the committed lock and contained cache. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-s7n-01 | Tampering / Elevation of Privilege | ZIP/tar.gz extraction | high | mitigate | Verify SHA-256 before inspection; reject traversal, duplicates, symlinks, hardlinks, devices/FIFOs/sockets/unknown types; extract regular files only into contained staging; ignore ownership and privilege bits. |
| T-s7n-02 | Tampering | install manifest no-op | high | mitigate | Require the new schema, exact archive hash, non-empty unique per-file hashes/modes, and live file verification; legacy/empty/malformed manifests force staged revalidation. |
| T-s7n-03 | Spoofing / Tampering | source URL and redirects | high | mitigate | Permit absolute local fixture files or committed allowlisted HTTPS origins only, re-check redirect destinations, reject HTTP/unknown schemes, and promote cache/install bytes only after exact checksum verification. |
| T-s7n-04 | Tampering | content-addressed download cache | high | mitigate | Re-hash cached bytes before reuse, discard only the corrupt exact cache entry, download to a partial file, verify, then atomically promote to the hash-derived cache name. |
| T-s7n-05 | Elevation of Privilege | POSIX archive modes | medium | mitigate | Preserve only ordinary permission bits needed for executability; never restore archive owner/group, setuid, or setgid; validate installed modes before a manifest match. |
| T-s7n-06 | Spoofing | platform/tool executable resolution | high | mitigate | Derive PlatformKey from runtime.GOOS/GOARCH, validate archive filename/root against the selected platform, use absolute pinned executable paths, and never consult PATH or derive missing pins. |
| T-s7n-07 | Tampering / Repudiation | go.mod/go.sum and npm locks | high | mitigate | Set readonly Go environment, snapshot raw bytes, compare on every exit, use exact npm ci, and emit stable mutation diagnostics before writing success manifests. |
| T-s7n-08 | Injection | child process invocation | high | mitigate | Use `exec.CommandContext` with argument vectors and explicit working directory/environment; never invoke through a shell or concatenate executable command text. |
| T-s7n-09 | Denial of Service | failed revalidation/promotion | medium | mitigate | Keep an existing install until a full replacement and manifest are staged, clean bounded partial/staging paths on failure, and modify only the exact requested tool install. |
| T-s7n-SC | Tampering | Go/npm package supply chain | high | mitigate | Add no package; retain exact go.mod/go.sum and package-lock pins, reuse the previously approved @linear/sdk 88.1.0 and TypeScript 7.0.2, disable npm lifecycle scripts/audit/fund side effects, and verify locks remain byte-identical. |
