<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local IPC client to Art-Net daemon | A local process submits strict framed command requests to the long-lived daemon; it must not become reachable from the network or from other local users by default. |
| shared Unix filesystem namespace to owner-only socket | `/tmp` is shared, so directory type, permissions, endpoint type, liveness, and cleanup decisions are untrusted inputs before the daemon listens. |
| build target to OS transport | Build tags must select exactly one transport and keep Windows-only `go-winio` symbols out of Linux/macOS compilation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-rq4-01 | Spoofing / Elevation of Privilege | `transport_unix.go` endpoint directory and socket | high | mitigate | Use a deterministic UID-scoped directory, reject symlink/non-directory parents, enforce/check 0700 on the directory and 0600 on the socket, and never introduce TCP. |
| T-rq4-02 | Tampering | stale-socket cleanup | high | mitigate | `Lstat` without following links, require `ModeSocket`, preserve an active listener, remove only after a refused/not-found probe, and test that regular files/symlinks remain intact. |
| T-rq4-03 | Denial of Service | stale or pre-created endpoint | medium | mitigate | Recover a verified stale socket deterministically while failing clearly on active or unsafe endpoint objects; close and clean a newly created listener if permission hardening fails. |
| T-rq4-04 | Information Disclosure | IPC socket access | high | mitigate | Retain owner-only Windows SDDL and enforce owner-only Unix directory/socket modes; preserve strict bounded framing and local-only transport. |
| T-rq4-SC | Tampering | dependency/build selection | low | accept | No package is installed, upgraded, or removed; the already-audited pinned `go-winio v0.6.2` remains isolated to Windows build files and the non-Windows implementation uses the Go standard library. |
</threat_model>
